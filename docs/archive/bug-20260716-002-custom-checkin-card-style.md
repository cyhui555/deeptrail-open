# BUG-20260716-002 自定义打卡点卡片样式热修摘要

- 状态：已关闭
- 日期：2026-07-16
- 版本：v0.1.1 热修候选，未改动 `v0.1.0` 标签

## 根因与修复

自定义点被地图选中后，现场页同时在外层卡片与内部 POI 卡应用高亮。内部高亮分支覆盖基础样式时漏掉白色背景，导致行程背景图穿透并形成双重蓝色描边，看起来像自定义点使用了另一套卡片。

高亮状态现统一由外层可交互卡片负责；内部 `PoiInfoCard` 始终保留与 AI 点一致的白色底、圆角、边框和内容层级。时段配色仍按上午、下午、晚上等真实语义区分。

第二次人工复核发现无描述的自定义项仍存在字段层级倒置：地图地址进入主内容色块，用户填写的内容却进入下方地点行。现通过共享展示映射统一两个入口：当自定义项无独立描述但存在地图地址时，主内容显示用户填写的名称/内容，下方地点行显示地图地址；AI 项和已填写描述的自定义项保持“描述在上、POI 在下”。现场页、完整路线与行程详情共用同一规则。

首次修复后，用户现有浏览器仍显示旧样式。进一步复现确认 3000 端口运行 `next dev`，但根布局无条件注册 PWA Service Worker；旧 Worker 对稳定的开发态 `/_next/static/` chunk 路径执行 cache-first，导致普通刷新继续返回旧组件。开发环境现会主动注销遗留 Worker、清理 `travel-traces` / `travel-planner-` 缓存并只自动重载一次；生产 Worker 升级到 v4，并在安装后立即 `skipWaiting`，避免旧版本长期等待接管。

## 验收

- Playwright 覆盖“AI 点与自定义点同底色、外层单次高亮”及两个入口的“内容在上、地点在下”，地图路线场景 4/4 通过。
- M13 浏览器边界 4/4 通过，其中新增“旧 Worker + 旧开发 chunk 缓存”真实生命周期回归；清理后 `controller=false`、注册数为 0、旅迹缓存数为 0。
- Web typecheck 与 lint 通过。
- 当前开发服务输出包含自动清理引导，`sw.js` 已为 v4 且包含 `skipWaiting`；真实浏览器按截图数据复核主内容为“玩”、地点行为“四川省成都市青羊区少城街道长顺下街66号”，页面无运行时错误。
