# ADR：单维护者 PR 允许人工管理员旁路

- 状态：已接受
- 日期：2026-07-19
- 关联：`REQ-GOV-002`、`TASK-GOV-004`

## 背景

工程实际只有一名 GitHub 维护者。Agent 编写和验证代码，但 GitHub App、CLI 或本地凭据会以工程所有者身份创建 PR；GitHub 因此把所有者同时视为 PR 作者，禁止其为自己的 PR 提交 `APPROVED` Review。强制作者外审批在该身份模型下形成不可满足的门禁。

机器人作者 Workflow 能提供账号级职责分离，但不覆盖所有普通 Agent 工作流，而且额外复制分支不能等价为第二位人员审计。工程所有者明确选择由自己审查 Agent 产出并承担最终合并决定。

## 决策

- 常规路径继续要求一项作者外 Review、五项 Required Checks、最新主干、线性历史和对话解决。
- `main` 对管理员不再强制整条经典保护规则；唯一工程所有者可在人工审查当前 Head 后显式执行管理员合并。
- 作者自己的 PR 不伪造 `APPROVED` Review；管理员合并事件与合并 Commit 是最终人工验收记录。
- 自动化不得调用 `--admin`、提交自审批、启用 auto-merge 或直推 `main`。L3B Engine 继续默认关闭，并继续拒绝管理员合并。
- 纯文档归档可按仓库协作规则由所有者授权后受检 fast-forward 直推；本 ADR 的配置与检查脚本由所有者一次性明确授权直接合入，不形成普通代码直推先例。

## 等价控制与风险

管理员在旁路前必须核对 PR Base/Head、精确 Head SHA、五项 Required Checks、Review 对话和变更范围。经典分支保护无法只对 Review 开放管理员旁路，因此平台同时允许管理员越过其他规则；这是单维护者模型的明确剩余风险，不得转授给 Agent 或 Workflow。

若未来增加第二名可靠维护者，应恢复 `enforce_admins=true`，让作者外审批重新成为平台强制条件。

## 回滚

恢复线上及仓库合同中的管理员强制保护，撤销本 ADR 的生效状态。已有 PR 与提交保留，不改写共享历史，也不触发部署。
