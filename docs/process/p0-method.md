# P0 工程方法与实时检查

## G0：开始前

- 关联 Requirement、Bug 或 Task ID。
- 明确目标、范围内、范围外和可验证验收标准。
- 读取 `docs/memory/project-state.md` 与相关设计。
- 列出测试、构建、Eval 和回滚方式。

关键项缺失时先补任务定义，不进入编码。

## G1：实施中

- 变更保持在工作项范围内。
- 不绕过应用、数据库、用户归属和外部数据校验边界。
- Bug 先有复现或失败证据，再修复。
- 迁移只修改目标工程，源工程保持只读。

## G2：提交前

- lint、typecheck、测试、构建和适用的 Eval 通过。
- 权限、日志脱敏、失败路径和密钥扫描通过。
- 配置、文档、Schema 与实际行为一致。

## G3：验收时

- 每条验收标准有命令、测试或运行时证据。
- 风险、限制和未验证项如实说明。
- Issue、看板、ExecPlan 与项目恢复点同步。

只有范围内实现与全部适用验证共同通过，工作项才可进入 Done。
