## 规划原则

- 先形成可回滚、可追踪的稳定发布基线，再扩展生产能力。
- 优先解决可能造成重复数据、隐私泄露或错误状态的问题。
- 新任务必须建立 Requirement、Work Item、验收标准和回滚方案后才能进入 `In Progress`。
- 默认继续使用本地 AI、地图和确定性数据替身；真实外部服务接入需单独确认权限与成本。

## M14：发布与数据可靠性

### ROADMAP-001：固化 M12/M13 发布基线

- 范围：复核工作树、敏感信息扫描、生成变更摘要、创建稳定提交并同步中央工程记忆。
- 验收：全量门禁保持通过；Git revision 可复现；中央校验 6/6，通过后 `last_result` 与实际同步状态一致。
- 交付：用户已明确授权封板；全量门禁通过后创建 `v0.1.0` 稳定 Git revision，并同步中央工程记忆。

### ROADMAP-002：离线同步幂等化

- 范围：为打卡和轨迹批次建立客户端幂等标识、服务端去重规则及数据库唯一约束；明确成功响应丢失后的重试语义。
- 验收：同一批次重复提交只产生一次结果；跨用户幂等键相互隔离；断线、超时和进程重启测试通过。
- 回滚：新增迁移只增加幂等字段或索引，不删除现有业务数据；旧客户端仍可使用受控兼容路径。
- 交付证据：[M14 离线同步幂等化交付摘要](../archive/m14-delivery.md)。

## M15：生产实时性与性能观测

### ROADMAP-003：多实例 SSE 可靠性

- 范围：抽象任务事件总线，评估 Redis Pub/Sub、Streams 或等价方案；处理断线重连、事件顺序和资源释放。
- 验收：任务在不同节点创建与订阅时仍能收到终态；事件总线不可用时自动回退轮询；连接数和错误率可观测。
- 依赖：需要确认生产部署拓扑和外部基础设施权限。

### ROADMAP-004：建立生产 RUM 基线

- 范围：在现有脱敏 Web Vitals 白名单上增加聚合、保留周期、路由分组基线和阈值告警，不采集 URL、用户标识或表单内容。
- 验收：至少形成首页、行程、现场和回忆四组 P75 指标；可区分版本回归；数据量、保留和删除策略有明确上限。
- 依赖：需要具备合规的生产指标样本和监控存储。

## M16：质量效率与离线体验

### ROADMAP-005：Playwright 稳定性专项

- 范围：优先清理审计发现的 15 处固定等待；复核 93 处位置选择器，只保留具有真实顺序语义的用法。
- 验收：不使用 `force`、`networkidle` 或任意固定等待；关键路径使用角色、标签或稳定业务状态；连续多轮回归无偶发失败。
- 顺序：先处理自定义地点编辑的 5 秒等待，再处理首页、日期摘要和地图场景。
- 交付证据：[M16 Playwright 稳定性专项交付摘要](../archive/m16-delivery.md)。

### ROADMAP-006：离线同步状态与失败恢复体验

- 范围：增加待同步数量、同步中状态、失败原因、手动重试和会话切换清理反馈；保持弱网下现场操作连续。
- 验收：用户可以判断数据是否已送达；失败记录不会静默丢失；退出和账号切换后不展示上一用户队列。
- 依赖：ROADMAP-002 已完成，当前依赖已满足。

## M17：产品闭环验证

### ROADMAP-007：旅行执行闭环验证

- 范围：验证“继续旅行”首页优先级、行程—现场—回忆三阶段导航、地图列表联动和复杂行程编辑效率。
- 验收：建立任务完成率、关键步骤耗时和失败点基线；每项产品改动均来自明确证据，并保留无障碍和移动端回归。
- 边界：本阶段先验证问题，不默认扩展票务、酒店、社交或公开路线市场。

## M18：后台账号分配

### ROADMAP-008：后台账号分配

- 范围：关闭默认环境公开注册；初始化管理员角色；提供普通用户分页查询、创建、启停和密码重置。
- 验收：生产注册入口不可用；普通用户不能访问后台；停用账号的登录与既有会话同时失效；桌面和 360px 浏览器回归通过。
- 交付证据：[后台用户管理交付摘要](../archive/task-ops-001-admin-user-management.md)。

## M19-M21：后台运营扩展

### ROADMAP-009：行程客服运营台

- 目标用户：处理用户反馈和任务失败的内部运营人员。
- 最小范围：按用户名、行程状态和时间检索；只读查看行程摘要、任务状态与脱敏错误；允许重试明确可重放的失败任务。
- 安全边界：默认不展示私有媒体原文件、精确轨迹或自由表单内容；数据修复必须单独确认并留痕。

### ROADMAP-010：运营审计与账号安全

- 最小范围：管理员敏感操作审计、管理员自助改密、首次登录强制改密、全部会话撤销和登录失败限流。
- 验收：可回答谁在何时对哪个账号执行了什么操作；审计记录不可由普通管理员修改；弱初始密码不进入长期运行状态。

### ROADMAP-011：AI 服务与用量观测

- 最小范围：按日聚合任务成功率、P50/P95 耗时、错误类型和 token 用量，提供 Provider 健康与降级状态。
- 隐私边界：不采集原始 Prompt、行程内容、用户标识或外部密钥；指标保留周期与删除规则有明确上限。

## 推荐执行顺序

后台运营建议顺序：`ROADMAP-008 → ROADMAP-010 → ROADMAP-009 → ROADMAP-011`。

先补齐审计和管理员改密，再开放更敏感的行程客服能力；AI 观测依赖脱敏指标口径。`ROADMAP-003` 与 `ROADMAP-004` 在生产部署和监控资源确认后启动；`ROADMAP-007` 可在获得真实可用性样本后独立立项。
