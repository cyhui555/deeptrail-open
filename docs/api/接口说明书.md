# API 接口入口

API 事实源是运行时 OpenAPI 与契约测试，本文件只保留接入约定，不手工复制全部请求/响应结构。

## 地址与契约

| 项目 | 地址或文件 |
| --- | --- |
| API 根地址 | `http://localhost:8080` |
| 健康检查 | `GET /api/health` |
| Swagger UI | `http://localhost:8080/swagger-ui.html` |
| OpenAPI JSON | `http://localhost:8080/v3/api-docs` |
| 最小契约基线 | `docs/api/openapi-contract-baseline.json` |
| 契约验证 | `pnpm test:contract` |

接口字段、状态码或 Schema 变化时，先修改 Controller/DTO 与测试，再更新 OpenAPI 基线；禁止只修改本说明。

## 通用约定

- 业务接口前缀为 `/api`，JSON 使用 UTF-8。
- 登录态通过 HttpOnly Cookie 传递；浏览器请求必须携带凭据，前端统一通过 `apps/web/src/lib/api.ts` 调用。
- 公开注册已关闭，账号只能由管理员通过后台分配；`test` Profile 的隔离注册夹具不属于生产契约。
- 所有资源读取和修改均在 Server 校验当前用户归属，不接受前端传入的用户 ID 作为授权事实。
- 日期时间使用 ISO-8601。行程表单允许 `yyyy-MM-ddTHH:mm`，服务端统一解析并返回明确时区语义。
- 分页参数从 1 开始，边界由 Controller 校验；非法参数返回 4xx，不交给数据库猜测。
- 离线打卡使用 `idempotencyKey`，轨迹点使用 `clientPointId`；标识由客户端首次提交前生成并在重试时稳定复用，旧客户端可省略。
- 异步 AI 任务由创建接口返回任务 ID，客户端轮询任务状态；失败必须返回可诊断错误，不得回退为伪造行程。

## 主要能力

| 能力 | 代表接口 |
| --- | --- |
| 认证 | `/api/auth/login`、`/api/auth/logout`、`/api/auth/me` |
| 后台用户管理 | `/api/admin/users`、`/api/admin/users/{userId}/status`、`/api/admin/users/{userId}/password`，仅 `ADMIN` |
| AI 状态 | `GET /api/ai/status` |
| 行程生成 | `/api/itineraries/generate`、`/optimize`、`/from-xiaohongshu` |
| 异步任务 | `GET /api/itineraries/tasks/{taskId}` |
| 行程管理 | `/api/trips`、`/api/trips/{planId}` |
| 打卡与轨迹 | `/api/trips/{planId}/checkin/*`、`/api/itineraries/checkin/*` |
| 媒体 | `GET /api/media/{mediaId}`，必须通过所有权和文件类型校验 |

完整方法、字段和响应以 Swagger/OpenAPI 为准。

## 错误与降级

- 4xx 表示请求、认证、归属或业务状态不满足；5xx 表示服务端异常。
- Web 不应把 API 不可达统一显示成“仍在加载”；超时后进入可重试错误态。
- 未配置 AI Provider 时 `/api/ai/status` 返回安全状态，生成接口明确不可用；不得返回测试桩数据。
- 地图配置或 SDK 加载失败时页面保留行程信息，并显示可诊断的地图降级提示。

## 变更检查

```powershell
pnpm test:contract
pnpm typecheck
pnpm test:e2e:smoke
```

涉及真实外部 Provider 的验证必须单独授权并记录范围；默认契约与 E2E 使用确定性替身。
